# 第二类：设备硬件规格 (Device Specifications)

这些参数定义了你模拟的是“哪一款手机”（如 iPhone 15 或华为 Mate 60）。

- **包含参数**：`C_nom_mAh` (电池电量)、`mass` (电池质量)、`area` (散热面积)、`V_min/V_max` (截止电压)。
    
- **获取方式**：
    
    1. **官方技术规格页**：去 Apple、Samsung 或华为官网查看产品参数。
        
    2. **第三方拆解报告**：如 **iFixit** 的拆解报告，会详细列出电池的重量 ($m$) 和具体尺寸（用于计算面积 $A$） 。
    
    3. **专业数据库**：如 **GSMArena**，可以查到几乎所有手机型号的标称电量和电压范围
```
  "device_specs": {

        "C_nom_mAh": 4500.0,   # 额定容量 [mAh]，典型智能手机 4000-5000 mAh

        "mass": 0.18,          # 电池质量 [kg]，约 180g

        "area": 0.015,         # 有效散热面积 [m²]，手机背部约 0.01-0.02

        "V_min": 3.0,          # 最低截止电压 [V]，低于此电压可能损坏电池

        "V_max": 4.2,          # 最高充电电压 [V]，锂电池标准上限

    },
```

# 第三类：组件功耗参数 (Component Power Parameters)

这些参数描述了屏幕、CPU、5G 等模块在工作时到底耗多少瓦电。

- **包含参数**：`P_base` (底噪功耗)、`k_scr` (屏幕系数)、`k_cpu` (运算系数)、`M_tech_factors` (4G/5G/WiFi 倍率)。
    
- **获取方式**：
    
    1. **开发者文档**：查阅 **Android Developer Guide** 或 **iOS Power Profile**。Google 的 Android 官方文档会提供不同硬件组件的典型功耗曲线 。

        
    2. **专业评测网站**：如 **AnandTech** 或 **NotebookCheck**。他们会对手机进行各组件的极端压力测试，并给出具体的功率（Watts）测量值 。
        
        
    3. **计算推导**：例如 `k_cpu` 可以通过设定“满载功耗为 5W”来反向推算系数 。
       
```
    # ------------------------------------------------------------------------

    # 功耗模型参数

    # ------------------------------------------------------------------------

    "power_params": {

        # 基础功耗

        "P_base": 0.4,         # 系统基础功耗 [W]，包括待机、传感器等

  

        # 屏幕功耗模型：P_screen = δ_on × (P_panel_base + k_scr × L^β)

        "P_panel_base": 0.5,   # 屏幕基础功耗 [W]，最低亮度时的功耗

        "k_scr": 2.2,          # 屏幕亮度系数 [W]

        "beta_screen": 1.4,    # 亮度非线性指数，>1 表示高亮度时功耗加速增长

  

        # CPU 功耗模型：P_cpu = P_leak × (1 + λ(T-T_ref)) + k_cpu × (α × η)³

        "P_leak_ref": 0.25,    # 参考漏电功耗 [W]，静态功耗基准

        "lambda_leak": 0.01,   # 漏电温度系数 [1/K]，温度每升高 1K 漏电增加 1%

        "k_cpu": 3.2,          # CPU 动态功耗系数 [W]

        "T_crit": 318.15,      # 热节流临界温度 [K]，即 45°C，超过后 CPU 降频

        "k_th": 0.25,          # 热节流 Sigmoid 斜率，控制降频的平滑程度

  

        # 网络功耗模型：P_net = M_tech × (P_idle + k_net × exp(-λ_snr × γ(t)))

        "P_idle": 0.35,        # 网络空闲功耗 [W]

        "k_net": 1.5,          # 网络功耗信号衰减系数 [W]

        "lambda_snr": 0.35,    # 信号强度衰减因子

  

        # 后台任务功耗（脉冲式）

        "P_wake": 0.25,        # 单次后台唤醒功耗 [W]

        "N_app": 6,            # 后台应用数量

        "bg_period": 60.0,     # 后台唤醒周期 [s]

        "bg_on": 5.0,          # 每次唤醒持续时间 [s]

  

        # GPS 功耗（脉冲式）

        "P_gps": 0.7,          # GPS 工作功耗 [W]

        "gps_period": 30.0,    # GPS 定位周期 [s]

        "gps_on": 10.0,        # 每次定位持续时间 [s]

  

        # 网络制式功耗倍增因子

        "M_tech_factors": {

            "4G": 1.0,         # 4G LTE 作为基准

            "5G": 1.35,        # 5G 功耗约高 35%

            "WiFi": 0.7,       # WiFi 功耗约低 30%

        },

    },
```

# 第四类：电化学与拟合参数 (Fitted Physical Parameters)

这是最体现“建模能力”的部分，描述了电池内部的化学特性。

- **包含参数**：`R_ref` (参考内阻)、`Ea` (活化能)、`ocv_coeffs` (开路电压系数)、`k_ref` (老化速率)。
    
- **获取方式**：
    
    1. **公开科研数据集**：题目明确提到可以使用公开数据集 。推荐搜索 **NASA Battery Dataset** 或 **Oxford Battery Degradation Dataset**。
        
    2. **学术论文**：在 Google Scholar 搜索关键词 “_Lithium-ion battery equivalent circuit model parameters_” 或 “_Battery aging rate Arrhenius equation_”。你会发现很多论文已经拟合好了这些常数，你只需引用即可 。

        
    3. **拟合计算**：使用 Python 的 `numpy.polyfit` 对放电曲线数据进行处理，得出 `ocv_coeffs` 。
        
```
    # ------------------------------------------------------------------------

    # 电池模型拟合参数（通常来自实验标定）

    # ------------------------------------------------------------------------

    "fitted_params": {

        "R_ref": 0.045,        # 参考内阻 [Ω]，T=T_ref 且 SOC=1 时的内阻

        "beta_pol": 0.012,     # 极化内阻系数 [Ω]，低 SOC 时内阻急剧上升的强度

        "k_ref": 1.2e-8,       # 老化速率基准 [1/(A·s)]，控制 SOH 下降速度

        "T_opt": 298.15,       # 最佳工作温度 [K]，远离此温度老化加速

        "k_soc": 1.5,          # SOC 极化指数，控制低 SOC 时极化内阻的陡峭程度

        "eps": 1e-4,           # 防除零小量，避免 SOC→0 时数值溢出

    },
```


```
 # ------------------------------------------------------------------------

    # OCV 多项式系数（5 阶，来自 NASA 电池数据集拟合）

    # V_oc(SOC) = c0 + c1×SOC + c2×SOC² + c3×SOC³ + c4×SOC⁴ + c5×SOC⁵

    # ------------------------------------------------------------------------

    "ocv_coeffs": [-12.13, 31.54, -28.75, 10.22, -0.88, 4.15],
```


# 第五类：策略与随机过程参数 (Control & Stochastic Parameters)

这些参数决定了系统的“行为习惯”和“随机波动”。

- **包含参数**：`T_crit` (热节流触发点)、`ou_params` (信号波动参数)。
    
- **获取方式**：
    
    1. **行业经验值**：例如手机外壳温度达到 $45^\circ C$ (318.15K) 通常会触发系统强制降频保护（Thermal Throttling），这是手机行业的普遍标准 。
        
        
    2. **建模假设**：对于信号强度波动（OU 过程），你可以设定均值为 0，通过灵敏度分析（Sensitivity Analysis）来观察不同的波动强度对续航的影响，这符合题目要求 。

```
      # ------------------------------------------------------------------------

    # OU 过程参数（用于模拟信号强度随机波动）

    # ------------------------------------------------------------------------

    "ou_params": {

        "mu": 0.0,             # 长期均值

        "theta": 0.15,         # 回归速度，越大波动衰减越快

        "sigma": 0.3,          # 噪声强度

    },

```
